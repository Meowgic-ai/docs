## 额度说明

额度用于在 AI 对话中生成、修改和优化电商应用。每次任务完成后，系统会根据任务复杂度动态计算并扣除相应额度。

通常任务越复杂，消耗的额度越多。

以下场景可能会消耗额度：

| 消耗类型 | 示例 |
| -------- | ---- |
| 需求理解 | 理解你的需求、拆解任务、规划应用结构 |
| 组件生成 | 生成按钮、表单、弹窗、导航菜单、商品卡片等电商组件 |
| 代码修改 | 调整样式、业务逻辑、交互行为或 API 集成 |
| 调试修复 | 修复报错、重新生成代码、调整应用结构 |
| 预览构建 | 生成可预览版本 |
| 第三方调用 | 调用外部服务或 API |

任务完成后，系统会按实际使用量扣除额度。

额度的扣减顺序为：剩余额度优先，其次月度额度，最后年度额度。这样可以在保障任务完成的同时，提高额度使用效率。

| 额度类型 | 说明 |
| -------- | ---- |
| 试用额度 | 仅在 7 天试用期内有效，试用期结束后会自动清零 |
| 月度额度 | 月付订阅生效后立即发放；每个新账期开始时会按订阅方案重置并重新发放 |
| 年度额度 | 按年付费；订阅后会先发放首月额度，后续额度按账期按月发放，共 12 次 |
| 剩余额度 | 上一账期未使用完的额度；切换方案后，原账期的剩余额度可继续使用且不过期 |

## 常见问题

<AccordionGroup>
  <Accordion title="有试用额度吗？">
    有。订阅 Business 方案后，系统会一次性发放试用额度。试用额度有效期为 7 天，过期后未使用部分会自动清零。
  </Accordion>
  <Accordion title="试用期内可以取消订阅吗？">
    可以。取消后，剩余试用额度会自动清零。
  </Accordion>
  <Accordion title="月度额度什么时候发放？">
    月付订阅生效后会立即发放额度，之后按你的账期每月发放。
  </Accordion>
  <Accordion title="年度额度是怎么发放的？">
    年度额度按月发放，共 12 次。订阅年付方案后会先发放首月额度，剩余额度按账期逐月发放。
  </Accordion>
  <Accordion title="切换方案后，原有额度还在吗？">
    在。已发放的剩余额度可继续使用且不过期；同时你也会拿到新方案对应的额度。
  </Accordion>
  <Accordion title="取消年付后，还会继续发放剩余年度额度吗？">
    会。取消年付只会关闭下一年的自动续费，不影响当前订阅周期。在当前年付周期内，剩余年度额度仍会按月继续发放，直到 12 次发放完成。
  </Accordion>
  <Accordion title="从年付切到月付后，额度会怎么变化？">
    原年付方案会立即结束，新月付方案会马上生效。由于原年付方案可能还有未发放完的月度分配，你在月付期间仍可能继续收到这部分年度额度。若月付方案也处于生效中，你将同时收到月度额度和剩余年度额度发放。
  </Accordion>
  <Accordion title="切换方案和取消订阅有什么区别？">
    取消订阅是关闭下一账期自动续费，当前订阅会持续到本账期结束；切换方案则是立即结束当前方案，并立刻启用新方案。
  </Accordion>
  <Accordion title="额度用完了怎么办？">
    你可以升级到更高额度的订阅方案。新方案生效后，对应额度会立即发放。
  </Accordion>
  <Accordion title="订阅后可以退款吗？">
    不支持。订阅生效并发放额度后，费用不予退还。你可以随时取消订阅，以停止后续自动续费。
  </Accordion>
  <Accordion title="如果任务消耗超过当前余额会怎样？">
    系统会先完整生成并展示任务结果，再扣除实际消耗额度。若扣除后余额为负，你将暂时无法发起新的 AI 对话任务，直到新额度发放。后续新发放的额度会先自动抵扣负余额。
  </Accordion>
</AccordionGroup>
